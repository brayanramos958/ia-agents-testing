ARQUITECTURA DEL AGENTE
--------------------------------------------------------------------------------

1. MEMORIA A CORTO PLAZO (Working Memory)

   - Gestionado por LangChain's ConversationBufferMemory
   - Mantiene los Гәltimos N mensajes en el contexto del LLM

   - El usuario puede corregir respuestas del agente
   - Las correcciones se almacenan en la tabla `corrections`

   - Se inyectan en el contexto del LLM para mejorar respuestas futuras

1. Usuario envГӯa mensaje вҶ’ Frontend React вҶ’ Backend API

3. Backend busca correcciones/aprendizajes relevantes en SQLite
4. Se construye prompt con: instrucciones base + historial + contexto de aprendizajes
5. LLM (Groq/Mixtral) genera respuesta
6. Respuesta almacenada en BD y enviada al frontend
7. Si usuario corrige: se guarda correcciГіn y se actualiza modelo de aprendizajes
