# Plan: ia-agent-tickets-v1

## Objetivo
Sistema de tickets para mesa de ayuda con agente IA (LangGraph) que asista en resoluciones.

---

## Estructura del Proyecto
```
ia-agent-tickets-v1/
├── backend/          (Express + SQLite)
└── frontend/         (React + Vite)
```

---

## Schema SQLite

### users
| Campo   | Tipo   | Descripción                    |
|---------|--------|--------------------------------|
| id      | INTEGER PRIMARY KEY | |
| nombre  | TEXT   | Nombre completo |
| cargo   | TEXT   | Puesto de trabajo |
| area    | TEXT   | Departamento |
| correo  | TEXT   | Email |
| rol     | TEXT   | creador / resolutor / supervisor |

### tickets
| Campo             | Tipo   | Descripción |
|-------------------|--------|-------------|
| id                | INTEGER PRIMARY KEY | |
| tipo_requerimiento | TEXT   | Tipo de ticket |
| categoria         | TEXT   | Categoría |
| estado            | TEXT   | abierto / asignado / resuelto / cerrado |
| asignado_a        | INTEGER FK | Usuario resolutor asignado |
| urgencia          | TEXT   | baja / media / alta / crítica |
| impacto           | TEXT   | bajo / medio / alto |
| prioridad         | TEXT   | baja / media / alta / urgente |
| resolucion        | TEXT   | Vacío (para agente después) |
| created_at        | DATETIME | |
| updated_at        | DATETIME | |

### ticket_history
| Campo     | Tipo   | Descripción |
|-----------|--------|-------------|
| id        | INTEGER PRIMARY KEY | |
| ticket_id | INTEGER FK | Ticket relacionado |
| accion    | TEXT   | creado / asignado / resuelto / reabierto |
| detalle   | TEXT   | Descripción del cambio |
| fecha     | DATETIME | |
| user_id   | INTEGER FK | Usuario que realizó la acción |

---

## Roles y Permisos

| Rol        | Acciones |
|------------|----------|
| Creador    | Crear tickets, ver mis tickets |
| Resueltor  | Ver tickets asignados a mí, marcar como resueltos |
| Supervisor | Ver TODOS los tickets, reasignar, reabrir |

---

## API Endpoints

### Users
- `GET /api/users` - Listar usuarios
- `GET /api/users/:id` - Ver usuario
- `GET /api/users?rol=creador` - Filtrar por rol

### Tickets
- `GET /api/tickets` - Listar tickets (con filtros)
- `POST /api/tickets` - Crear ticket (creador)
- `GET /api/tickets/:id` - Ver ticket
- `PUT /api/tickets/:id` - Actualizar ticket
- `PUT /api/tickets/:id/assign` - Asignar ticket (supervisor)
- `PUT /api/tickets/:id/resolve` - Resolver ticket (resueltor)
- `PUT /api/tickets/:id/reopen` - Reabrir ticket (supervisor)

### History
- `GET /api/tickets/:id/history` - Historial del ticket
